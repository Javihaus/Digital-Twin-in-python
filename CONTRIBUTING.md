# Contributing to otwin

Thanks for your interest in improving otwin. The project values **correctness and
rigor over breadth** — a small, well-tested core beats a large, shaky one.

## Development setup

```bash
git clone https://github.com/Javihaus/otwin.git
cd otwin
python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -e ".[dev]"
pytest
```

## Standards (all enforced in CI)

- **Typing:** `mypy --strict` must pass (the package ships `py.typed`).
- **Style:** `ruff check` and `black --check` must pass.
- **Tests:** `pytest` green; core-module coverage gate is **85%**.
- **Structural guarantees** are covered by property-based tests where applicable
  (e.g. PHS `J` skew / `R` PSD, energy non-increase, calibration coverage).
- **No swallowed failures** in CI.

Run everything locally before opening a PR:

```bash
ruff check src/otwin && black --check src/otwin tests
mypy --strict src/otwin
pytest --cov=otwin --cov-fail-under=85
```

## No unverified claims

Do not assert a reference, formula, or empirical number as fact unless it is
checked. New citations go in `CITATIONS.md` with a `VERIFIED` / `UNVERIFIED`
status; benchmark numbers must be generated by code (see `benchmarks/`), never
hand-typed.

## Scope

- Core (`numpy` + `scipy` only) stays dependency-light.
- Heavier capabilities (`torch`, `gp`, visualization, optimization) live behind
  optional extras and in `examples/`.

## Reporting issues

Open a GitHub issue with a minimal reproduction, your platform, and the otwin
version (`python -c "import otwin; print(otwin.__version__)"`).
